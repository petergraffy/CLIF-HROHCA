# Federated Pooling Plan

## Site-Level Workflow

Each CLIF site runs the same project scripts locally against its own CLIF tables and local exposure inputs.

Required site outputs:

- `*_dlnm_site_estimates.csv`
- `*_dlnm_time_sensitivity.csv`
- `*_table1.csv`
- `*_outcomes.csv`
- `*_adverse_outcome_models.csv`
- `*_continuous_outcome_models.csv`
- `*_adverse_outcome_rates.csv`

These files are generated by:

```r
Rscript code/run_site_analysis.R
```

The export step writes aggregate-only files to `output/final/federated_exports/`. Sites should not share `output/intermediate/` or any other local CLIF-derived working files.

## Shared Estimate

The primary pooled effect is the log relative risk for the 95th percentile `tmax` contrast versus the site-specific warm-season median `tmax`.

Primary model:

- negative-binomial DLNM
- warm season only
- same-day county-linked `tmax`
- humidity adjusted
- median reference
- lag `0-5`

Sensitivity models:

- humidity plus county-year `no2` and `pm25`
- MRT reference
- calendar-time spline sensitivity using 3, 4, and 6 df per year
- day-of-week exclusion sensitivity
- subgroup-specific estimates by age, sex, and race

Secondary patient-level outcome models estimate whether same-day heat exposure among OHCA ICU patients is associated with:

- death or discharge to hospice
- hospital death
- invasive mechanical ventilation
- vasopressor use
- ICU length of stay
- invasive mechanical ventilation duration among ventilated patients

## Coordinating Center Workflow

The coordinating center places all site `*_dlnm_site_estimates.csv` files in:

```text
output/final/federated_exports
```

Then run:

```r
Rscript code/90_pool_federated_results.R
```

This writes:

- `output/final/federated_pooled/all_site_dlnm_estimates.csv`
- `output/final/federated_pooled/pooled_dlnm_random_effects_results.csv`

## Current Pooling Method

The current pooling script uses DerSimonian-Laird random-effects meta-analysis on scalar site-level log relative risks.

This is intentionally simple and easy for all sites to export. A more advanced second-stage DLNM pooling approach can be added later by exporting reduced crossbasis coefficients and variance-covariance matrices from each site.
